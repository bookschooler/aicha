# 제품 요구사항 정의서 (PRD) - 상권 분석 서비스 "BlueOcean Finder"

## 1. 프로젝트 개요
사용자가 특정 주소를 입력하면 해당 주소가 속한 상권을 식별하고, 해당 상권의 블루오션 지수(수요 대비 공급 부족도), 예상 매출액, 유동인구 성격, 점포 수 등을 종합적으로 분석하여 창업 및 입지 결정에 도움을 주는 웹 서비스입니다.

## 2. 목표 및 핵심 가치
- **데이터 기반 의사결정**: 단순 직관이 아닌 9분기 분량의 서울시 상권 데이터를 기반으로 한 정밀 분석 제공.
- **블루오션 발굴**: "수요는 충분하나 공급(찻집)이 적은" 기회 시장을 발굴하고, **전체 상권 중 해당 입지의 순위(Ranking)**를 명확히 제시.
- **직관적인 UI**: 주소 검색 한 번으로 복잡한 상권 지표와 **정확한 순위**를 한눈에 파악.

## 3. 타겟 사용자
- 서울 지역 내 찻집(카페 포함) 창업을 준비 중인 예비 창업자
- 기존 사업장의 입지 경쟁력을 평가하고 싶은 자영업자
- 상권 분석 데이터를 필요로 하는 컨설턴트 및 투자자

## 4. 주요 기능 (Functional Requirements)

### F1. 주소 기반 상권 매핑
- 사용자가 도로명/지번 주소를 입력하면 서울시 1,139개 상권 영역 중 어디에 속하는지 매핑.
- 해당 상권의 유형(골목상권, 발달상권, 전통시장 등) 노출.

### F2. 상권 랭킹 및 블루오션 지수
- **입지 순위 표시**: 서울시 전체 1,139개 상권 중 해당 주소지가 속한 상권의 **블루오션 통합 순위(1위~1,139위)**를 메인 대시보드 상단에 노출.
- **블루오션 지수**: 분석 모델(OLS 잔차)과 공급 지수를 결합한 사분면 분류(Q1~Q4) 결과 및 해당 사분면 내에서의 상세 등수 제공.
- **백분위 정보**: "상위 5% 입지", "추천 순위 상위 10%" 등 상대적 위치 정보를 시각적 게이지로 표시.

### F3. 매출 및 수요 예측
- **예상 매출액**: 6가지 주요 수요 변수(집객시설, 직장인구, 소득, 가구수, 검색지수, 지하철 노선)를 바탕으로 해당 상권의 월평균 예상 매출액 제시.
- **수요 요인 분석**: 매출에 가장 큰 영향을 미치는 핵심 요인(SHAP 값 기반) 가시화.

### F4. 상세 상권 지표 (Dashboard)
- **유동인구 분석**: 총 유동인구, 직장인구, 상주 인구 데이터 제공.
- **공급 현황**: 해당 상권 내 점포 수, 카페/음료 점포 대비 찻집 비율 분석.
- **소비 수준**: 월평균 소득 금액 및 가구 수 정보 제공.

## 5. 비기능 Requirements
- **반응성**: 주소 검색 시 2초 이내 분석 결과 대시보드 출력.
- **시각화**: 레이더 차트, 바 차트 등을 활용하여 상권의 강점/약점을 시각적으로 비교 가능하게 함.
- **데이터 최신성**: 분기별 데이터 업데이트 파이프라인 구축.

## 6. 기술 스택 및 아키텍처 제안
- **Back-end**: Python (FastAPI) - 기존 분석 스크립트(.py)와 데이터 모델을 API화하여 서빙.
- **Front-end**: React + Vite + TailwindCSS - 빠르고 세련된 대시보드 UI 구현.
- **Data**: CSV 기반 데이터베이스 또는 SQLite (경량화 서비스 형태).
- **Map**: Kakao Maps API 또는 Naver Maps API (주소-좌표 변환 및 영역 표시용).

## 7. 향후 확장성
- 찻집 외 다른 업종(치킨집, 편의점 등)으로 모델 확장.
- 전국 상권 데이터로 범위 확대.
- 머신러닝 모델(XGBoost)을 실시간 추론 엔진으로 고도화.
