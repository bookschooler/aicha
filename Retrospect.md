| 접근성 | 지하철역수, 지하철노선수 | 직접 수집·계산 |

### 조합지표 생성 수식

```python
공급갭_지수   = 카페음료_점포수 / (찻집_수 + 1)
유동밀도_지수 = 여성_유동인구_수 / (찻집_수 + 1)
상주밀도_지수 = 여성_상주인구_수 / (찻집_수 + 1)
여가소비_지수 = 여가_지출_총금액 / (찻집_수 + 1)
```

> **설계 의도:** 찻집 1개당 잠재 수요를 나타내는 밀도 지표.
> 찻집이 적을수록 값이 커지게 설계 → 블루오션 신호 포착 목적.

## ❓ Q1. XGBoost+SHAP 이후 추가로 해볼 분석이 뭐가 있나? 특정 상권에만 통하는 feature는 어떻게 살릴까?

> 이런 context-specific feature를 살리는 방법이 필요.

**제안된 추가 분석 목록:**

| 분석 | 목적 |
|------|------|
| SHAP Interaction Values | feature 간 조합 효과 수치화 |
| 상권 유형별 세그먼트 분리 SHAP | 어느 유형에서 어떤 feature가 중요한지 |
| Lasso/ElasticNet | 최종 변수 자동 선별 |
| 패널 데이터 처리 | Fixed/Random Effects |

**context-specific feature 처리 방안 비교:**

| 방법 | 장점 | 단점 |

| 세그먼트별 별도 모델 | 유형별 최적화 | 잔차 스케일 달라져 전체 비교 불가 |

## ❓ Q2. 프로젝트 분석 구조 자체에 문제가 없나? (방법론 재검토)

> **배경:** "예측매출 vs 실제매출로 블루오션 발굴"이라는 최초 방법론을 더 깊이 검토.

### 🚨 발견된 3가지 구조적 문제

---

### 문제 1 — Y변수 함정 (가장 중요)

```
진짜 블루오션:  유동인구 높음 + 찻집 0개  →  Y = 0
레드오션:       유동인구 높음 + 찻집 20개  →  Y = 높음
```

**핵심:** Y(매출)는 수요만 반영하는 게 아니라 **공급이 있어야 발생**.
→ "예측 > 실제 = 블루오션"이 **"찻집 적음 = 블루오션"이라는 동어반복**이 될 위험.

---

### 문제 2 — 공급변수를 X에 넣으면 잔차 해석 붕괴

```
X에 카페음료_점포수, 찻집_수 포함 시:
  잔차 = "비슷한 공급 수준 대비 성과 차이"
       ≠ "잠재수요 대비 공급 부족"  (우리가 원하는 것)
```

---

### 문제 3 — 조합지표 이중계산

```python
# 이 변수들을 X에 넣으면서 찻집_수도 따로 X에 넣으면 → 찻집수 이중 투입
유동밀도_지수 = 여성유동인구 / (찻집수 + 1)
상주밀도_지수 = 여성상주인구 / (찻집수 + 1)
여가소비_지수 = 여가지출총금액 / (찻집수 + 1)
공급갭_지수   = 카페음료점포수 / (찻집수 + 1)
```

## ✅ 결정 1 — 2-Track 분리 프레임워크 채택

```
┌──────────────────────────────────────────────────┐
│  Track A: 순수 수요 예측 모델                     │
│    X = 수요지표만 (공급변수 완전 제외)             │
│    Y = log(매출)                                  │
│    → OOF 잔차 추출 (수요 대비 저성과 측정)        │
└──────────────────────────────────────────────────┘
                        +
┌──────────────────────────────────────────────────┐
│  Track B: 공급 부족 지수 (모델 밖에서 계산)       │
│    공급부족지수 = 1 / (찻집_수 + 1)              │
│    보조: 공급갭_지수 = 카페음료점포수/(찻집수+1)  │
└──────────────────────────────────────────────────┘
```

## ❓ Q3. 2-Track 가중치를 어떻게 결정할까?

> 앤트로피 가중치법 vs 대리변수 회귀법

### 앤트로피 가중치법 검토

- **가정:** 분산이 큰 지표 = 더 차별적 = 더 중요
- **문제:** 이 가정이 블루오션 목적에 맞는지 불명확. 결과가 데이터 구조에 따라 달라짐.

### 대리변수 회귀법 검토

- **아이디어:** `카페_개업률`을 블루오션 proxy로 사용
- **문제:** 개업률 높음 = 이미 기회를 발견한 곳 = 오히려 블루오션 소멸 중.
  논리 순환 + 찻집 전용 미래 개업 데이터 없음.

### ✅ 결정 2 — 2D 매트릭스 방식 채택

```
X축: Track A OOF 잔차 (음수 = 수요 대비 저성과)
Y축: 공급부족지수 = 1/(찻집_수 + 1)

         │ 공급부족 높음
         │    (찻집 적음)
   ②  ★  │  ①
         │
─────────┼──────────── 잔차
         │         (양수=과성과)
   ③     │  ④
         │
```

## ❓ Q5. SHAP만 보고 변수를 선택해도 되나, XGBoost 중요도도 같이 봐야 하나?

| SHAP 평균 절대값 | 실제 예측값 이동량 | 상관 변수 간 기여 분산 |

### 불일치 해석 기준

```
XGB 순위 > SHAP 순위  →  XGB 과대평가.  SHAP을 믿어라.
SHAP 순위 > XGB 순위  →  비선형 효과.  SHAP을 믿어라.
둘 다 상위            →  확실한 핵심 변수.
둘 다 하위            →  확실한 제거 후보.
```

### 실제 사례

| 변수 | XGB 순위 | SHAP 순위 | 판정 |

## ❓ Q6. 패널 데이터(상권×분기) 구조 어떻게 처리할까?

> Fixed Effects를 쓰면 상권별 고유 특성이 흡수 → 상권 간 비교 불가.

### 옵션 비교

| 방식 | 장점 | 단점 |
|------|------|------|
| Entity FE (상권 FE) | 관찰 불가 특성 통제 | 상권 고유 특성 제거됨 → 비교 불가 |
| Time FE만 (분기 더미) | 계절성 통제 + 상권 특성 유지 | 상권 간 누락변수 편향 잔존 |
| Pooled OLS (더미 없음) | 단순 | 계절성 혼재 |

### ✅ 결정 5 — Pooled OLS + 분기 더미 + 상권유형 더미

```
log(매출)_it = α + Σβ_j·X_j_it + Σγ_t·Q_t + Σδ_k·Type_k + ε_it

Q_t   : 분기 더미 8개 (20233 기준, 20234~20253)
Type_k: 상권유형 더미 (관광특구/발달상권/골목상권/전통시장)
```

| 효과 | 설명 |
|------|------|
| 계절성 제거 | Q3 여름/Q4 겨울 공통 충격 흡수 |
| 경기 공통 충격 제거 | 2024 소비 위축 등 전체 트렌드 통제 |
| 상권 간 비교 가능 | 개별 상권 FE 없음 → 상권 특성이 계수에 살아있음 |
| 유형 간 수준 통제 | 같은 유형 내에서 잔차 비교 가능 |

## ❓ Q8. OOF 추출 시 K-Fold 분할 기준은 무엇으로 해야 하나?

### 옵션 비교

| 방식 | 설명 | 문제점 |
|------|------|--------|
| Random KFold | 일반 랜덤 분할 | 같은 상권의 다른 분기가 train/test 동시 포함 → data leakage |
| **GroupKFold** | 상권 코드 기준 분할 | 같은 상권 9분기 통째로 분리 → 정직한 검증 ★ |
| TimeSeriesSplit | 시간 순서 분할 | 미래 예측 목적에 적합. 우리 목적과는 다소 다름 |

### ✅ 결정 6 — GroupKFold(groups=상권_코드, n_splits=5)

> **이유:** 블루오션 목적은 "새 상권 예측"이 아니라 **기존 상권들의 잔차 비교**.
> 같은 상권의 시계열 누수를 막는 것이 핵심.

```

```

```
실제 찻집 없음 → 실제 매출: 낮음
잔차 = 낮은 실제 - 높은 예측 = 음수

해석: "수요 조건은 충분한데 찻집이 없어서 수요가 아직 실현 안 됨"
```

```
Q2 = 유동인구 많은데 편의점이 없는 거리 → 아직 아무도 열지 않은 것
Q1 = 유동인구 적어 보이는데 편의점이 잘 되고 있는 거리 → 의외로 잘 되는 것
```

**리스크 차이:**
- Q2: 수요지표 기반 추정 → 찻집 수요가 실제로 발현될지 미지수 (고위험)

## ❓ Q24. 공급 지표를 하나만 쓴 것 같아서 논리적 기반이 약하다. 찻집 관련 요소도 없는 것 같다. 어떻게 방어할건가?

> "그런데 그 공급 지표라는 게 한 가지만 써서 좀 논리적으로 기반이 약한 것 같습니다.
> 공급지표가 카페음료 점포당 매출인데 여기에 찻집에 관한 요소는 전혀 없어서 좀 정확도가 떨어져 보입니다."

### ✅ 결정 18 — 방어 논리 정리

**즉각 반박:**
1. 공급 지수는 단일 지표가 아닌 **복합 지수(2개 요소)**
   - 0.5 × rank(1/(찻집수+1))               ← 찻집 수 직접 반영
   - 0.5 × rank(찻집매출/(카페음료점포수+1)) ← 분자가 찻집 매출
2. "찻집 고객은 목적성이 강하다"는 주장은 오히려 분석의 강점
   → 수요가 목적성 기반일수록 공급 공백의 신호가 더 의미 있음

- 앤트로피 가중치법, 대리변수 회귀법 모두 검토했으나 블루오션 목적에 더 자의적이어서 포기 (Retrospect Q3 참조)
