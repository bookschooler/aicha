## 1부 — 문제 정의

### S1. 표지
- **제목:** 서울 찻집 창업을 위한 블루오션 상권 분석
- **부제:** 수요-공급 갭 기반 데이터 추천 시스템
- 소속 / 이름 / 날짜

---

### S2. 왜 찻집인가?
- 카페 시장 포화 → 틈새 업종으로 찻집 주목
- 기존 접근법의 한계: "매출 높은 곳 = 좋은 곳" → 이미 경쟁 포화
- **핵심 질문:** 서울 1,139개 상권 중 *수요는 있는데 찻집이 없는 곳*은 어디인가?

> **비주얼 제안:** 카페 vs 찻집 점포수 비교 수치 (단순 텍스트 강조)

---

### S3. 데이터 소개
- **규모:** 서울시 1,139개 상권 × 9분기 (2023Q3 ~ 2025Q3) = 9,760행 / 155개 변수
- **수집 방법:**

| 구분 | 내용 | 출처 |
|------|------|------|
| 수요 | 인구·소득·집객시설·검색트렌드 | 서울시 상권분석서비스 API / 네이버 데이터랩 |
| 공급 | 찻집 수·매출, 카페 점포수 | 카카오 API 직접 크롤링 |
| 접근성 | 지하철 노선 수 | 직접 수집 후 좌표 매핑 |

## 2부 — 방법론

### S4. 분석 프레임워크 전체 조감
```
┌──────────────────────────────────────────┐
│  수요 변수 6개 (SHAP → Lasso → VIF 확정)  │
│         ↓                                 │
│  Track A: Pooled OLS + GroupKFold        │
│         → OOF 잔차                        │
│           양수 = 수요 대비 과성과          │
│           음수 = 수요 대비 저성과(미실현)  │
└──────────────────────────────────────────┘
             +
┌──────────────────────────────────────────┐
│  Track B: 복합 공급부족 지수              │
│  0.5 × rank(1/(찻집수+1))               │
│  0.5 × rank(찻집매출/(카페점포수+1))     │
└──────────────────────────────────────────┘
             ↓
┌──────────────────────────────────────────┐
│  2D 매트릭스 → Q1 / Q2 사분면 → 추천    │
└──────────────────────────────────────────┘
```

> **포인트:** 한 장에 전체 흐름 다이어그램으로 표현

---

### S5. Track A — 수요 예측 모델
- **왜 공급변수를 제외했나?**
  - Y(찻집 매출)는 공급이 있어야 발생 → 공급변수 포함 시 잔차 해석 붕괴
  - 잔차가 "찻집 적음"을 이미 포착하면 Track B가 중복됨
  - Track A = "커피·음료 시장이 얼마나 실현됐나"

- **최종 수요 변수 6개:**

| 변수 | SHAP 순위 | VIF | 의미 |
|------|---------|-----|------|
| 집객시설_수 | 2위 | 2.89 | 수요-인프라 |
| 총_직장_인구_수 | 3위 | 1.72 | 인구 대표 |
| 월_평균_소득_금액 | 4위 | 3.28 | 소비력 |
| 총_가구_수 | 7위 | 2.37 | 상주 소비층 |
| 카페_검색지수 | 9위 | 1.62 | 트렌드 |
| 지하철_노선_수 | 17위 | 1.74 | 접근성 |

- **모델 성능:** OOF R² = 0.4413 → 낮은 게 의도적 (높으면 잔차 신호 소멸)

---

### S6. Track B — 복합 공급부족 지수
- **두 요소의 역할:**
  - `1/(찻집수+1)` → 찻집이 0개일수록 높은 점수 (직접적 공급 부재)
  - `찻집매출/(카페점포수+1)` → 카페 경쟁 속에서의 찻집 성과 (시장 내 위치)
- **rank 변환 후 0.5/0.5:** 두 지표 스케일 동일 → 각 차원 동등 고려

> **포인트:** 두 요소를 나란히 시각화 + "왜 둘 다 필요한가" 한 줄 강조

---

### S7. 변수 선택 과정
```
전체 155개 변수
      ↓  VIF 사전 정리 (다중공선성 제거)
     22개 대표 변수
      ↓  XGBoost + SHAP (비선형 중요도)
     상위 후보 추출
      ↓  Lasso / ElasticNet (정규화 회귀)
      ↓  VIF 재확인
     최종 6개 수요 변수
```

> **비주얼 제안:** 깔때기(funnel) 다이어그램 또는 단계별 숫자 강조
